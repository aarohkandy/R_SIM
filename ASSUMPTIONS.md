# Assumptions

Every placeholder or stub below must be replaced or reaffirmed by later phases before its
dependent model is treated as real engineering data.

| ID | Area | Assumption | Replacement path |
| --- | --- | --- | --- |
| A-0001 | Inputs | Real motor `.eng`/`.rse` data is not present yet; `inputs/motor_D21_placeholder.eng` is a labeled placeholder. | Replace with measured or vendor motor data in `inputs/`, then update `config/motor.yaml`. |
| A-0002 | Inputs | Real vehicle BOM/mass/material data is not present yet; placeholder YAML files exercise mass, depletion, and deployable-leg behavior only. | Replace with measured BOM and material properties. |
| A-0003 | Inputs | Real OpenRocket frozen-configuration exports are not present yet; `inputs/openrocket/frozen_placeholder.csv` is a labeled comparison placeholder. | Add exported CP/Cd anchors under `inputs/openrocket/` and update `config/aero.yaml`. |
| A-0004 | Firmware | Real ESP32 and Teensy firmware ELFs are not present yet. | Add firmware sources/ELFs under `firmware/` before Phase 12. |
| A-0005 | Tooling | Renode, CalculiX/Gmsh, and ffmpeg are not required for Phase 0 and may be absent locally. | Later phases should attempt documented installs or stub/log blockers per `SPEC.md`. |
| A-0006 | Commands | `make sensitivity` and `make soak` are exposed but intentionally fail until their phases implement them. `make e2e`, `make converge`, and `make montecarlo` are implemented as of Phases 8, 13, and 14. | Replace remaining command stubs during Phases 15 and 17. |
| A-0007 | Cold gas | The Phase 5 CO2 model uses real-gas CoolProp properties, but tank volume, shell thermal mass, heat-transfer rate, regulator setpoint/sag, and nozzle throat areas are placeholder hardware values. | Replace with bench-measured cartridge/regulator/nozzle data and update `config/coldgas.yaml`; keep golden tests only after deliberate re-baselining. |
| A-0008 | Dynamics | Phase 6 dynamics settings for motor thrust axis, thrust application point, minimum aero speed, and angular damping are placeholder values used to exercise the plant. | Replace with measured thrust-line/alignment data, damping identification, and validated geometry before using trajectory outputs as engineering evidence. |
| A-0009 | Sensors | Phase 7 IMU/barometer sample rates, noise densities, bias/random-walk terms, scale factors, misalignment, saturation, and lag are placeholder calibration values. | Replace with chosen sensor datasheets, Allan-variance logs, bench calibration, and flight-computer sample-rate measurements; deliberately re-baseline sensor goldens afterward. |
| A-0010 | Control/actuation | Phase 8 SIL PD gains, landing-burn gate, latency/jitter values, solenoid timing, and allocation scaling are placeholder values that make the controller seam and e2e run executable; they are not tuned flight gains. | Replace with controller design/tuning evidence, valve bench timing, measured firmware loop timing, and hardware-in-the-loop comparison data; deliberately re-baseline SIL goldens afterward. |
| A-0011 | IO/animation | `ffmpeg` is not installed locally, so Phase 9 emits animated GIF and interactive HTML artifacts and records MP4 export as deferred in the run manifest. | Install `ffmpeg` before requiring MP4 export, then enable and verify the MP4 writer without changing the telemetry source. |
| A-0012 | Thermal | Phase 10 thermal masses, convection coefficients, conductances, emissivity/view factor, motor casing heat power, solenoid self-heat, and electronics self-heat are placeholder values used to exercise the post-flight thermal network. | Replace with measured motor casing temperature data, material coupons, geometry-derived areas, bench heat-transfer tests, solenoid/electronics power measurements, and updated material limits; deliberately re-baseline Phase 10 goldens afterward. |
| A-0013 | Structural | CalculiX/Gmsh/FEniCS are not installed locally, so Phase 11 exports a CalculiX input deck and runs the configured internal deterministic linear-truss FEA fallback. | A-0014 | Renode HIL | Phase 12 has a Backend-B preflight harness, sensor/actuator exchange shapes, a dual-MCU Renode script, and placeholder ESP32/Teensy platform files, but local Renode, `pyrenode3`, real firmware ELFs, and verified board `.repl` models are absent. | Install Renode and the Python control bridge, add real `firmware/esp32_flight.elf` and `firmware/teensy_flight.elf`, replace placeholder `.repl` files with verified board/peripheral models, then rerun `make hil` and proceed to real firmware co-sim. |
| A-0015 | Cross-validation | RocketPy is not installed locally, so Phase 13 emits analytic ballistic and OpenRocket-anchor cross-validation and documents RocketPy comparison as unavailable rather than faking a passive-ascent match. | Install RocketPy and add a configured passive-ascent scenario adapter, then rerun `make converge` and record RocketPy deltas as data. |
| A-0016 | Monte Carlo | Phase 14 now has a real native-SIL Monte Carlo runner and deterministic scenario generator, but only smoke/pilot runs should be claimed unless the configured 1000-run target and percentile-stability criteria have actually completed. | Run `make montecarlo` without the smoke override, let all configured runs finish, inspect `outputs/phase14_montecarlo/montecarlo_summary.json`, and record whether the percentile distributions reached stability. |
